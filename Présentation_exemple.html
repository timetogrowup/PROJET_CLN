Présentation d'exemple d'un projet hybride

Contexte du Projet

Dans ce projet, nous utilisons des requêtes SQL pour extraire des données depuis un datastore, que nous utilisons ensuite pour créer un fichier historique de référence. Ce fichier est mis à jour automatiquement chaque jour, et les données stockées servent de base pour une API qui est utilisée par une application.

Fonctionnement

Le processus se déroule en plusieurs étapes :

  • Extraction des données : Une requête SQL est utilisée pour extraire des données à partir du datastore, ce qui permet de collecter les informations nécessaires.
  • Mise à jour quotidienne : Le fichier historique de référence est mis à jour automatiquement tous les jours à l'aide d'une routine planifiée.
  • Exploitation de l'API : L'API est conçue pour utiliser ces données historiques, permettant à une application d'interroger directement les informations stockées.

Solutions techniques utilisées

Ce projet s'appuie sur une combinaison de deux solutions :

  • Pipeline de données : Les données sont extraites, transformées et chargées dans un fichier historique de référence, ce qui correspond à une approche pipeline.
  • Intégration directe via API : L'API exposée permet à une application d'accéder directement aux données mises à jour quotidiennement.

Bénéfices de cette approche

Cette approche permet de garantir la mise à jour continue des données tout en offrant une solution flexible et évolutive à travers une API. En combinant les solutions de pipeline et d'intégration via API, le projet assure des performances optimales pour répondre aux besoins de l'application consommatrice des données.
